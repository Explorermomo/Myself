Core Problem

Placement order is a hidden control knob

Modern ML placers usually optimize spatial coordinates, while the temporal decision of macro sequence is fixed by simple priors such as large-first or dense-first.

Early mistakes cascade. In a sequential placer, the first macros define feasible boundary conditions for everything that follows.

Naive search is expensive. Fully evaluating each candidate order requires a complete placement cycle.

Theory

Bad order can asymptotically hurt HPWL

OrderPlace reframes placement sequencing as a first-class optimization variable: avoid bad early commitments instead of repairing them after the fact.

Search Object

Code-level ordering policies

Each strategy defines a priority score over remaining macros using geometry, net topology, and current placement context.

φ : M × C → Rt,   sort remaining macros by ascending priority
  • Static seeds: area, degree, area→degree, net-area.
  • Dynamic seeds: spring, field, entropy, Hamiltonian.
  • LLM variants: gravity, adaptive clustering, net closure, phase-aware entropy.
End-to-End Check

OpenROAD PPA trends carry through

  • ChipBench validation: lower HPWL and congestion on all evaluated designs.
  • Routed wirelength: lower than EfficientPlace on all four OpenROAD cases.
  • Physical quality: smaller area on all cases; WNS/TNS/NVP improve on three cases.
Method

OrderPlace: proxy-guided LLM evolution

Overview adapted from Figure 1: LLMs generate placement-order strategies, a proxy evaluator ranks them, and elite policies guide the next generation.
Step 1
Seed
Start from static and dynamic human priors so the LLM sees useful ordering patterns.
Step 2
Evolve
Prompt the LLM with top-K strategies, scores, source code, and current best policy.
Step 3
Place
Use wire-mask-guided greedy placement; the evolved order controls global trajectory.
